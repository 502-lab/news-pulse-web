# Feature Specification: 003 메인 대시보드 (W-01)

**Feature Branch**: `003-main-dashboard`

**Created**: 2026-06-12

**Status**: Draft

## User Scenarios & Testing

### User Story 1 — 통계 현황 한눈에 파악 (Priority: P1)

사용자가 대시보드에 진입하면 오늘의 뉴스 수, 분석된 기사 수, 평균 편향 점수, 모니터링 중인 키워드 수를 StatCard 4종으로 즉시 확인한다. 전일 대비 증감도 함께 표시되어 변화 추이를 직관적으로 파악한다.

**Why this priority**: 서비스의 핵심 현황 데이터를 즉시 파악하는 것이 대시보드의 1차 목적. 다른 기능 없이도 단독으로 의미 있는 MVP를 제공한다.

**Independent Test**: 대시보드 진입 시 StatCard 4종 렌더링만으로 완결적인 테스트 가능.

**Acceptance Scenarios**:

1. **Given** 사용자가 대시보드(`/`)에 진입했을 때, **When** 페이지가 로드되면, **Then** 오늘의 뉴스 수 / 분석된 기사 수 / 평균 편향 점수 / 모니터링 중인 키워드 수 카드 4개가 표시된다.
2. **Given** 데이터 로딩 중일 때, **When** StatCard 영역을 보면, **Then** shimmer 애니메이션 스켈레톤 UI가 표시된다.
3. **Given** 데이터 로드 완료 후, **When** 각 카드를 보면, **Then** 수치 + 전일 대비 delta(% 및 방향 화살표)가 표시된다.
4. **Given** delta가 양수일 때, **When** 카드를 보면, **Then** 증감 표시가 초록색이다.
5. **Given** delta가 음수일 때, **When** 카드를 보면, **Then** 증감 표시가 빨간색이다.
6. **Given** delta가 0일 때, **When** 카드를 보면, **Then** 증감 표시가 회색이다.
7. **Given** 데이터 조회 실패 시, **When** StatCard 영역을 보면, **Then** "데이터를 불러올 수 없습니다." 에러 메시지가 표시된다.

---

### User Story 2 — 카테고리별 뉴스 분포 확인 (Priority: P2)

사용자가 카테고리 바차트를 통해 정치 / 경제 / 사회 / 기술 / 문화 / 스포츠 6개 카테고리의 오늘 뉴스 수 분포를 시각적으로 파악한다.

**Why this priority**: StatCard 이후 두 번째 핵심 요약 정보. 단독으로 카테고리 분포를 시각화하여 독립적 가치를 제공한다.

**Independent Test**: 바차트 컴포넌트만 렌더링해도 카테고리 분포 확인 가능.

**Acceptance Scenarios**:

1. **Given** 대시보드에 진입하면, **When** 차트 영역을 보면, **Then** 6개 카테고리의 막대 차트가 표시된다.
2. **Given** 데이터 로딩 중, **When** 차트 영역을 보면, **Then** shimmer 스켈레톤이 표시된다.
3. **Given** 데이터 조회 실패, **When** 차트 영역을 보면, **Then** "데이터를 불러올 수 없습니다." + 재시도 버튼이 표시된다.
4. **Given** 재시도 버튼 클릭, **When** 클릭하면, **Then** 데이터를 다시 조회한다.
5. **Given** 모든 카테고리 count가 0, **When** 차트 영역을 보면, **Then** "아직 집계된 데이터가 없습니다." + 빈 차트 영역이 표시된다.

---

### User Story 3 — 최신 뉴스 목록 탐색 (Priority: P3)

사용자가 뉴스 피드에서 최신 뉴스 10개를 확인하고, 관심 기사를 클릭해 상세 페이지로 이동한다.

**Why this priority**: 통계/차트 이후의 보완 정보. 단독 렌더링으로 최신 뉴스 열람이라는 독립적 가치를 제공한다.

**Independent Test**: 뉴스 피드 컴포넌트만 렌더링해도 뉴스 탐색 가능.

**Acceptance Scenarios**:

1. **Given** 대시보드에 진입하면, **When** 뉴스 피드를 보면, **Then** 최신 뉴스 10개가 표시된다.
2. **Given** 각 뉴스 카드를 보면, **When** 렌더링되면, **Then** 썸네일 + 제목 + 출처 + 카테고리 배지 + 발행 시간(상대 시간)이 표시된다.
3. **Given** 썸네일 URL이 null/undefined일 때, **When** 카드를 보면, **Then** 회색 fallback 박스 + 신문 아이콘이 표시된다.
4. **Given** 뉴스 카드를 클릭하면, **When** 클릭하면, **Then** `/articles/:id`로 이동한다.
5. **Given** 데이터 로딩 중, **When** 피드를 보면, **Then** shimmer 스켈레톤 10개가 표시된다.
6. **Given** 데이터 조회 실패, **When** 피드를 보면, **Then** "뉴스를 불러올 수 없습니다." + 재시도 버튼이 표시된다.
7. **Given** 뉴스 항목이 없을 때, **When** 피드를 보면, **Then** "표시할 뉴스가 없습니다." + 아이콘이 표시된다.

---

### Edge Cases

- 썸네일 이미지 로드 실패 시(네트워크 오류): fallback 박스로 대체
- 발행 시간이 미래인 경우: 상대 시간 표시 (예: "방금 전")
- 카테고리 배지에 매핑되지 않는 카테고리 값: 기본 배지 스타일로 표시
- 뉴스 제목이 매우 긴 경우: 줄임표(ellipsis) 처리

## Requirements

### Functional Requirements

- **FR-001**: 대시보드 진입 시 StatCard 4종(오늘의 뉴스 수 / 분석된 기사 수 / 평균 편향 점수 / 모니터링 중인 키워드 수)을 표시해야 한다.
- **FR-002**: 각 StatCard는 수치 + 전일 대비 delta 값 + deltaPercent + 방향 화살표를 표시해야 한다.
- **FR-003**: delta 부호에 따라 색상을 구분해야 한다 (양수: 초록, 음수: 빨강, 0: 회색).
- **FR-004**: 카테고리별 뉴스 수를 막대 차트로 표시해야 한다 (6개 카테고리: 정치/경제/사회/기술/문화/스포츠).
- **FR-005**: 뉴스 피드는 최신 뉴스 10개를 카드 형태로 표시해야 한다.
- **FR-006**: 각 뉴스 카드는 썸네일 + 제목 + 출처 + 카테고리 배지 + 상대 시간을 포함해야 한다.
- **FR-007**: 썸네일 URL이 없거나 이미지 로드 실패 시 fallback UI(회색 박스 + 아이콘)를 표시해야 한다.
- **FR-008**: 뉴스 카드 클릭 시 해당 기사 상세 페이지(`/articles/:id`)로 이동해야 한다.
- **FR-009**: 모든 데이터 영역은 로딩 중 shimmer 스켈레톤 UI를 표시해야 한다.
- **FR-010**: 데이터 조회 실패 시 에러 메시지와 재시도 버튼을 표시해야 한다.
- **FR-011**: 재시도 버튼 클릭 시 해당 데이터 조회를 재시도해야 한다.
- **FR-012**: 뉴스 피드 항목이 없을 때 빈 상태 메시지를 표시해야 한다.
- **FR-013**: 카테고리 차트 데이터가 모두 0일 때 빈 상태 메시지를 표시해야 한다.

### Key Entities

- **DashboardStat**: 단일 통계 카드 데이터 — 레이블, 아이콘, 수치, 전일 대비 증감값, 증감율
- **CategoryData**: 카테고리별 뉴스 집계 — 카테고리명, 오늘의 기사 수
- **NewsItem**: 뉴스 목록 항목 — ID, 제목, 출처, 카테고리, 발행 시간, 썸네일 URL

## Success Criteria

### Measurable Outcomes

- **SC-001**: 대시보드 진입 후 500ms 이내에 스켈레톤 UI가 표시되고, 데이터 로드 완료 시 실제 데이터로 전환된다.
- **SC-002**: 3개 데이터 영역(StatCard, 바차트, 뉴스 피드) 각각이 독립적으로 에러/로딩/성공 상태를 처리한다.
- **SC-003**: 뉴스 카드 클릭 후 1초 이내에 상세 페이지로 이동한다.
- **SC-004**: 썸네일 없는 뉴스 항목에서 레이아웃 깨짐 없이 fallback UI가 표시된다.
- **SC-005**: 에러 발생 시 재시도 버튼 클릭으로 해당 영역만 독립 재조회된다.

## Assumptions

- 003 단계는 mock 데이터로 구현하며, 실제 API 연동은 이후 스프린트에서 처리한다.
- mock은 500ms 딜레이를 포함해 스켈레톤 UI 동작을 검증한다.
- 페이지네이션 없이 고정 10개 목록만 표시한다 (v1 스코프).
- 카테고리 배지 색상은 `constants/category.ts`에서 관리하는 기존 규칙을 따른다.
- 접근성(aria 속성) 준수는 모든 컴포넌트에 필수 적용된다.
- 다크/라이트 모드 대응은 Tailwind `dark:` 클래스로 구현한다.
- 북마크 기능, 검색, 인기 뉴스 등 W-01의 다른 하위 기능은 이후 스프린트 대상이다.
